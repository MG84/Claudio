# Phase 2: Chronos-Bolt Predictions

> Source: docs/TRADING_ARCHITECTURE.md — Step 2

## Goal

Add Chronos-Bolt (Amazon) as a second forecasting signal alongside Kronos. Provides univariate close-price predictions with native uncertainty bands (quantile forecasts), giving Claude richer information for decision-making.

## Context

From the architecture analysis:
- Kronos: specialist OHLC, trained on financial data → primary signal
- Chronos-Bolt: generalist, 250x faster than original Chronos, CPU-friendly, pure PyTorch
- If both models agree on direction → higher confidence
- If they disagree → caution, reduce position or skip
- Chronos-Bolt adds uncertainty bands that Kronos lacks: "90% probability between $X and $Y"

## Tasks

### Task 1: Create `bot/chronos_predictor.py`
Module following the same pattern as `bot/kronos.py`:

- `init()` — load `amazon/chronos-bolt-small` via `transformers` AutoModelForSeq2SeqLM (cached in Docker volume)
- `is_ready()` — public readiness check
- `predict(closes: list[float], horizon: int = 12)` → dict with:
  - `point_forecast`: list of predicted close prices
  - `quantile_forecast`: dict with `q10`, `q50`, `q90` lists
  - `direction`: "UP" or "DOWN"
  - `change_pct`: predicted % change
- `format_prediction(result)` — Telegram-formatted string
- `chronos_loop()` — background task, predict every hour for each configured pair

Implementation notes:
- Use `ChronosPipeline.from_pretrained("amazon/chronos-bolt-small")` from `chronos-forecasting` package
- Input: torch tensor of close prices (last N candles)
- Output: quantile forecasts via `pipeline.predict(context, prediction_length, quantile_levels=[0.1, 0.5, 0.9])`
- Use `asyncio.to_thread()` for blocking inference
- Emit `chronos_prediction` event via `monitor.emit()` for dashboard
- Store predictions in `kronos.db` in a separate `chronos_predictions` table (same schema pattern)
- Disableable with `CHRONOS_ENABLED=false`

### Task 2: Add config constants
Add to `bot/config.py`:

```python
# ── Chronos-Bolt ────────────────────────────────────
CHRONOS_ENABLED = os.getenv("CHRONOS_ENABLED", "true").lower() == "true"
CHRONOS_MODEL_NAME = "amazon/chronos-bolt-small"
CHRONOS_PRED_LEN = 12
CHRONOS_LOOKBACK = 400
CHRONOS_INTERVAL_SECONDS = 3600
CHRONOS_DB_PATH = MEMORY_DIR / "kronos.db"  # shared DB, separate table
```

### Task 3: Add dependency
Add `chronos-forecasting>=1.0.0` to `requirements.txt` (or just use `transformers` directly if the package isn't needed).

### Task 4: Wire into `bot/main.py`
- Import and call `init()` in startup
- `asyncio.create_task(chronos_loop())` as background task
- Follow same pattern as Kronos wiring

## Verification

- `predict([95000, 95100, 95200, ...], horizon=12)` returns dict with point_forecast (12 values), quantile_forecast (q10/q50/q90 each 12 values)
- `format_prediction()` returns readable Telegram string with direction, %, and bands
- Background loop emits `chronos_prediction` events every hour
- Inference completes in <60s on CPU ARM
- Disabling with `CHRONOS_ENABLED=false` skips init and loop
