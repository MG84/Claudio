# Phase 3: Execution Layer + Kraken CLI

> Source: docs/TRADING_ARCHITECTURE.md — Step 3

## Goal

Trading execution wrapper around Kraken CLI with hard-coded risk limits that Claude cannot bypass. The safety architecture principle: **limits in CODE, not in prompt.**

## Context

From the architecture:
```
Claude says: "buy 50% of portfolio"
    ↓
trading.py: risk_check() → DENIED (max 20% per position)
    ↓
Claude receives: "Trade rejected: exceeds max position size (20%)"
```

Kraken CLI has paper trading built-in (`kraken paper init/buy/sell/status`) — zero API keys needed for testing.

## Tasks

### Task 1: Install Kraken CLI in Dockerfile
Add to Dockerfile:
- Download pre-built binary for linux/arm64 from GitHub releases
- Place in `/usr/local/bin/kraken`
- Make executable

Note: Check https://github.com/krakenfx/kraken-cli/releases for latest arm64 release URL.

### Task 2: Init paper trading in entrypoint
Add to `scripts/entrypoint.sh`:
- `kraken paper init --balance 10000` (if not already initialized)
- Only run if `TRADING_ENABLED=true`

### Task 3: Create `bot/trading.py`
Wrapper around Kraken CLI with risk manager:

**Public functions:**
- `place_order(side, pair, type, volume, price?, stop_loss?, take_profit?)` — risk check + execute via Kraken CLI
- `cancel_order(order_id)`
- `get_balance()` → portfolio dict
- `get_positions()` → list of open positions
- `get_trade_history(limit?)` → recent trades
- `risk_check(side, pair, volume, stop_loss)` → `{ok: bool, reason: str}`
- `get_daily_pnl()` → P&L for today
- `emergency_close_all()` → kill switch, close all positions
- `log_trade_rationale(trade_id, reasoning)` — save Claude's reasoning for the decision

**Risk checks (hard-coded, non-bypassable):**
- Max position size: `MAX_POSITION_PCT` (20%) of portfolio
- Max open positions: `MAX_OPEN_POSITIONS` (3)
- Max daily loss: `MAX_DAILY_LOSS_PCT` (5%) → stop trading for the day
- Max drawdown: `MAX_DRAWDOWN_PCT` (15%) → kill switch
- Stop-loss required: `STOP_LOSS_REQUIRED` (True)
- Max trades per day: `MAX_TRADES_PER_DAY` (10)

**Implementation:**
- Invoke Kraken CLI via `asyncio.create_subprocess_exec()`
- Parse JSON output
- SQLite `trades.db` for trade journal (separate from kronos.db)
- Mode: `paper` (default) or `live`
- Every `place_order()` call: risk_check → log → execute → log result

### Task 4: Add trading config constants
Add to `bot/config.py`:

```python
# ── Trading safety limits ────────────────────────────
TRADING_ENABLED = os.getenv("TRADING_ENABLED", "true").lower() == "true"
TRADING_MODE = os.getenv("TRADING_MODE", "paper")
TRADING_AUTONOMOUS = False
MAX_POSITION_PCT = 0.20
MAX_OPEN_POSITIONS = 3
MAX_DAILY_LOSS_PCT = 0.05
MAX_DRAWDOWN_PCT = 0.15
STOP_LOSS_REQUIRED = True
MAX_TRADES_PER_DAY = 10
TRADES_DB_PATH = MEMORY_DIR / "trades.db"
```

## Verification

- `risk_check("buy", "BTC/USDT", volume_exceeding_20pct)` returns `{ok: False, reason: "exceeds max position size"}`
- `risk_check("buy", "BTC/USDT", valid_volume, stop_loss=None)` returns `{ok: False, reason: "stop-loss required"}`
- `place_order()` in paper mode executes via `kraken paper buy` and logs to trades.db
- `emergency_close_all()` closes all open positions
- `get_balance()` returns current paper portfolio
- Every trade has a row in trades.db with timestamp, params, outcome
