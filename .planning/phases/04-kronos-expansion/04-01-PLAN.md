# Phase 4: Kronos Expansion

> Source: docs/TRADING_ARCHITECTURE.md — Step 4

## Goal

Extend the existing `bot/kronos.py` module with multi-pair support, instant access to latest predictions, and confidence scoring based on historical accuracy.

## Context

`bot/kronos.py` currently:
- Predicts only BTC/USDT
- Runs inference every hour via `kronos_loop()`
- Stores predictions in SQLite with verification
- Has `predict()`, `verify_predictions()`, `get_accuracy_stats()`, `format_prediction()`, `format_accuracy()`

Needs:
- `get_latest_prediction()` — for Claude to check without waiting for inference
- `predict_pair(pair)` — for any pair, not just hardcoded BTC/USDT
- `get_prediction_confidence()` — based on historical accuracy by market type

## Tasks

### Task 1: Add `get_latest_prediction(pair?)`
Query the predictions table for the most recent prediction, optionally filtered by pair. Return dict with current_price, predictions, created_at, or None if no predictions exist.

### Task 2: Add `predict_pair(pair, timeframe?)`
Generalize the existing `predict()` function to accept any pair. The current `predict()` uses hardcoded `KRONOS_SYMBOL` — refactor to accept pair as parameter, defaulting to config value.

### Task 3: Add `get_prediction_confidence(pair?)`
Calculate confidence score based on:
- Direction accuracy over last N predictions (higher accuracy = higher confidence)
- Recent MAE trend (decreasing = higher confidence)
- Return dict with `confidence: float (0-1)`, `based_on: int` (number of verified predictions), `direction_accuracy: float`

### Task 4: Update `kronos_loop()` for multi-pair
If `TRADING_PAIRS` is configured, loop over each pair (not just `KRONOS_SYMBOL`). Maintain backward compatibility — if only `KRONOS_SYMBOL` is set, behave as before.

## Verification

- `get_latest_prediction()` returns most recent prediction dict or None
- `get_latest_prediction("ETH/USDT")` filters by pair
- `predict_pair("ETH/USDT")` fetches ETH candles and runs Kronos inference
- `get_prediction_confidence()` returns confidence dict with score 0-1
- Existing `/predict` and `/accuracy` commands still work unchanged
