# Phase 1: Market Data + Indicators

> Source: docs/TRADING_ARCHITECTURE.md — Step 1

## Goal

Centralized market data aggregator for any crypto pair, with technical indicators computed on the fly. This module provides the "eyes" for Claude — all market context in one place.

## Context

Claudio already fetches OHLCV data in `bot/kronos.py` via ccxt for Kronos predictions. This module extracts and generalizes that pattern into a reusable market data service for any pair/timeframe, adding technical indicator calculations.

## Tasks

### Task 1: Create `bot/market.py` core module
Create the market data module with these functions:

- `get_ohlcv(pair, timeframe, limit)` — fetch OHLCV from Binance via ccxt (async)
- `get_ticker(pair)` — current price, 24h volume, 24h change %
- `get_orderbook(pair, depth)` — bid/ask spread
- `get_indicators(pair, timeframe)` — compute RSI(14), EMA(20/50/200), MACD(12/26/9), Bollinger(20,2), ATR(14) from OHLCV data
- `get_market_summary(pairs)` — formatted text snapshot for Claude's context (all pairs, all indicators)

Implementation notes:
- Use `pandas-ta` for indicator calculations
- Cache results with configurable TTL (default 60s for OHLCV, 30s for ticker)
- Use `asyncio.to_thread()` for CPU-bound indicator calculations
- Follow singleton pattern from `bot/kronos.py` (module-level state, `_get_exchange()`)
- Default pairs: configurable via `TRADING_PAIRS` in config

### Task 2: Add config constants
Add to `bot/config.py`:

```python
# ── Market data ─────────────────────────────────────
TRADING_PAIRS = ["BTC/USDT", "ETH/USDT", "SOL/USDT"]
MARKET_CACHE_SECONDS = 60
MARKET_TICKER_CACHE_SECONDS = 30
MARKET_OHLCV_LIMIT = 400
```

### Task 3: Add pandas-ta dependency
Add `pandas-ta>=0.3.0` to `requirements.txt`.

### Task 4: Test market data functions
Verify `get_ohlcv()`, `get_ticker()`, and `get_indicators()` return expected data structures. Test cache behavior (second call within TTL returns cached data).

## Verification

- `get_ohlcv("BTC/USDT", "1h", 100)` returns list of 100 candles with [timestamp, open, high, low, close, volume]
- `get_indicators("BTC/USDT", "1h")` returns dict with rsi, ema20, ema50, ema200, macd, bollinger, atr keys
- `get_market_summary(["BTC/USDT"])` returns formatted string suitable for Claude context
- Cache works: two rapid calls to `get_ohlcv` produce only one exchange API call
