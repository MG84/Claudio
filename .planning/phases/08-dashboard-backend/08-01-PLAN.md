<task type="auto">
  <title>Fix auth token to be deterministic</title>
  <files>
    - bot/ws_server.py
  </files>
  <action>
    Replace the current `_generate_auth_token()` function which uses `secrets.token_hex(8)` (random, regenerates on every restart) with a deterministic hash derived from the password:

    ```python
    import hashlib

    def _generate_auth_token() -> str:
        if not DASHBOARD_PASSWORD:
            return ""
        return hashlib.sha256(f"claudio:{DASHBOARD_PASSWORD}".encode()).hexdigest()
    ```

    This ensures the auth token is stable across restarts — same password always produces the same token, so browser cookies remain valid.

    Remove the `secrets` import if it was only used for this function.
  </action>
  <verify>
    <automated>grep -n "hashlib.sha256" bot/ws_server.py | head -1 && echo "PASS: deterministic token" || echo "FAIL: no hashlib usage found"</automated>
  </verify>
  <done>Dashboard session cookie survives bot restarts without requiring re-login (same password produces same token)</done>
