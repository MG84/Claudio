# Phase 8, Plan 2: REST Endpoints + CORS + Deploy Script

Add the 4 REST endpoints the trading dashboard frontend needs for initial data loading, CORS headers for dev mode, deploy script, and documentation updates.
Source: `docs/TRADING_DASHBOARD.md` — Wave 0, tasks 4-6 + blind spot #1.

## Goal

The frontend developer can start building the trading dashboard with hot-reload: REST endpoints return real data, CORS allows cross-origin requests from the dev server, and a deploy script automates the build-copy-rebuild cycle.

## Tasks

<task type="auto">
  <title>Add 4 REST endpoints + CORS to ws_server.py</title>
  <files>
    - bot/ws_server.py
  </files>
  <action>
    Add 4 new authenticated endpoints and CORS support.

    **CORS headers:** Add middleware or response headers to allow requests from `http://localhost:3000` (Next.js dev server). Add `Access-Control-Allow-Origin`, `Access-Control-Allow-Methods`, `Access-Control-Allow-Headers`, `Access-Control-Allow-Credentials` headers. Can use `aiohttp-cors` or manual `@middleware` — prefer manual middleware to avoid a new dependency.

    **GET /api/market/{pair}/{timeframe}:**
    - Convert pair from URL format (dash) to ccxt format (slash): `BTC-USDT` -> `BTC/USDT`
    - Call `bot.market.get_ohlcv(pair, timeframe)` -> OHLCV array
    - Call `bot.market.get_ticker(pair)` -> ticker dict
    - Call `bot.market.get_indicators(pair, timeframe)` -> indicator scalars
    - Return JSON: `{ "pair", "timeframe", "ohlcv", "ticker", "indicators" }`
    - Handle errors (invalid pair, exchange down) with 400/500 + error message

    **GET /api/portfolio:**
    - Call `bot.trading.get_balance()` -> balance dict
    - Call `bot.trading.get_positions()` -> positions list
    - Call `bot.trading.get_daily_pnl()` -> P&L dict
    - Call `bot.trading.get_risk_status()` -> risk dict
    - Return JSON: `{ "mode", "balance_usd", "initial_balance", "positions", "daily_pnl", "risk" }`

    **GET /api/trades?limit=20:**
    - Read `limit` query param (default 20, max 100)
    - Call `bot.trading.get_trade_history(limit)` -> trades list
    - Return JSON: `{ "trades": [...] }`

    **GET /api/chronos:**
    - Query `chronos_predictions` table in `kronos.db` directly (same pattern as existing `_kronos_handler()`)
    - Return JSON: `{ "predictions": [...] }`

    Register all routes in `start_server()`:
    ```python
    app.router.add_get("/api/market/{pair}/{timeframe}", _market_handler)
    app.router.add_get("/api/portfolio", _portfolio_handler)
    app.router.add_get("/api/trades", _trades_handler)
    app.router.add_get("/api/chronos", _chronos_pred_handler)
    ```

    All endpoints must be protected by `_check_auth()` (same as existing endpoints).

  <title>Create deploy dashboard script</title>
  <files>
    - scripts/deploy_dashboard.sh (new)
  </files>
  <action>
    Create `scripts/deploy_dashboard.sh` that builds claudio-monitor and deploys to Claudio in one command:

    ```bash
    #!/bin/bash
    # Deploy dashboard from claudio-monitor to Claudio
    set -e
    MONITOR_DIR="${CLAUDIO_MONITOR_DIR:-$HOME/Documents/Development/claudio-monitor}"
    CLAUDIO_DIR="$(cd "$(dirname "$0")/.." && pwd)"

    if [ ! -d "$MONITOR_DIR" ]; then
        echo "Error: claudio-monitor not found at $MONITOR_DIR"
        echo "Set CLAUDIO_MONITOR_DIR to override."
        exit 1
    fi

    echo "Building claudio-monitor..."
    cd "$MONITOR_DIR" && npm run build

    echo "Copying static export..."
    rm -rf "$CLAUDIO_DIR/dashboard/"*
    cp -r out/* "$CLAUDIO_DIR/dashboard/"

    echo "Rebuilding Docker..."
    cd "$CLAUDIO_DIR" && docker compose down && docker compose up -d --build

    echo "Deploy complete."
    ```

    Make executable: `chmod +x scripts/deploy_dashboard.sh`
  </action>
