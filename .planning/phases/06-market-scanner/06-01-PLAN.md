# Phase 6: Market Scanner + Risk Monitor

> Source: docs/TRADING_ARCHITECTURE.md — Steps 7-8

## Goal

Background loops that give Claudio autonomous market analysis capability and continuous risk monitoring. The scanner assembles context and lets Claude reason; the risk monitor enforces hard limits.

## Tasks

### Task 1: Market scanner loop
Background task (every hour, after Kronos + Chronos-Bolt predictions):

1. Get latest Kronos prediction via `kronos.get_latest_prediction()`
2. Get latest Chronos-Bolt prediction via `chronos_predictor.get_latest_prediction()`
3. Get market data + indicators via `market.get_market_summary(TRADING_PAIRS)`
4. Get portfolio state via `trading.get_balance()` + `trading.get_positions()`
5. Get risk status via `trading.get_daily_pnl()`
6. Assemble everything into a structured context block
7. If `TRADING_AUTONOMOUS=True` and signal is strong enough:
   - Pass context to Claude via bridge, Claude decides trade
   - Risk check in code before execution
   - Emit `trade_executed` event
8. If `TRADING_AUTONOMOUS=False`:
   - Send market brief to Telegram: "Market update: BTC $95,200 (+1.5%). Kronos: UP +1.2%. Chronos: UP +0.8% (concordi). RSI 45, trend bullish. Suggerisco: [Claude's recommendation]"
   - Wait for user response if action proposed

Event-driven triggers (in addition to hourly):
- Price change > 3% in 15 minutes → immediate scan
- Open position P&L crosses -2% → alert

### Task 2: Risk monitor loop
Background task (every 5 minutes):

1. Check all open positions vs stop-loss levels
2. Calculate current drawdown from peak portfolio value
3. Check daily P&L vs `MAX_DAILY_LOSS_PCT`
4. If daily loss exceeded → stop trading for the day, notify Telegram
5. If drawdown exceeded → `emergency_close_all()`, notify Telegram urgently
6. Emit `risk_alert` event if any limit is approaching (>80% of threshold)
7. Emit `portfolio_update` event with current state

### Task 3: Wire into `bot/main.py`
- Create background tasks for scanner and risk monitor
- Scanner should wait for Kronos and Chronos init before starting
- Risk monitor starts immediately

## Verification

- Scanner runs every hour and produces market brief in Telegram
- Risk monitor detects when daily loss exceeds 5% and stops trading
- Risk monitor detects drawdown >15% and calls emergency_close_all()
- Event-driven trigger fires on 3%+ price move
- `risk_alert` events appear in dashboard
- Scanner context includes all available data (predictions, indicators, portfolio)
